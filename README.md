# Git/GitHub CLIコマンドリファレンス

## Git基本コマンド

### 初期設定

**`git init`**
- ローカルフォルダをGitリポジトリとして初期化
- ⚠️ **重要**: 必ずプロジェクトのルートディレクトリで実行すること
  - ホームディレクトリなどで実行すると、意図しないファイルまで追跡対象になる危険性あり

**`git remote add origin <リポジトリのURL/SSH>`**
- リモートリポジトリを`origin`という名前で登録
- 以降のpush/pullコマンドで`origin`として参照可能

---

### 日常的な作業フロー

**`git add .`**
- カレントディレクトリ配下の変更をステージングエリアに追加
- `.`はカレントディレクトリを意味する
- 特定ファイルのみ追加する場合: `git add <ファイル名>`

**`git commit`**
- ステージングされた変更をコミットとして記録

主なオプション:
- `-m "メッセージ"` - コミットメッセージを直接指定
- `--amend` - 直前のコミットを修正（メッセージや内容の変更）
- `-a` - 追跡済みファイルを自動的にステージング（`git add`不要）
- `-v` - 変更内容を確認しながらコミットメッセージを編集
- `-p` - 変更箇所ごとにインタラクティブにステージング
- `--no-edit` - コミットメッセージを変更せずに実行（`--amend`と併用）

**`git push origin master`**
- ローカルの変更をリモートの`master`ブランチにプッシュ
- 現在は`main`ブランチが主流: `git push origin main`

## GitHub CLI (gh)コマンド

### リポジトリ管理

**`gh repo create`**
- GitHub上に新しいリポジトリを作成
- インタラクティブに設定を選択可能

**`gh repo edit`**
- リポジトリの設定を編集（説明、トピック、可視性など）

### プルリクエスト

**`gh pr create`**
- 新しいプルリクエストを作成
- ブランチ、タイトル、説明などを指定

**`gh pr list`**
- プルリクエストの一覧を表示
- オプションで絞り込みも可能

**`gh pr merge <PR番号>`**
- 指定したプルリクエストをマージ
- マージ方法（merge/squash/rebase）を選択可能

### Issue管理

**`gh issue create`**
- 新しいIssueを作成
- タイトル、本文、ラベルなどを設定

## よく使う組み合わせ

```bash
# 基本的な変更のプッシュ
git add .
git commit -m "変更内容の説明"
git push origin main

# 直前のコミットメッセージを修正
git commit --amend -m "修正後のメッセージ"
git push origin main --force

# 変更内容を確認しながらコミット
git add .
git commit -v
```

## Tips

- コミットメッセージは明確に書く（何を、なぜ変更したか）
- 頻繁に小さなコミットを作る方が、後で管理しやすい
- `git status`で現在の状態を確認する習慣をつける
- プッシュ前に`git log`で履歴を確認
